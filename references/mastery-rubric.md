# 掌握度六级量表（0-5）

## 量表定义

| 等级 | 名称 | 判定标准 | 关键标识 |
|------|------|----------|----------|
| 0 | 未接触 | 对该内容完全无了解 | 无法给出任何有效回答 |
| 1 | 识别 | 能选出正确答案，不能解释 | 选择题可以做对，但"为什么"说不清 |
| 2 | 理解 | 能用自己的话清晰解释 | 可以脱离术语说清核心逻辑 |
| 3 | 应用 | 能举例、类比、对比、在不同情境中使用 | 给出的例子不是书上的原例 |
| 4 | 分析 | 能拆解逻辑结构、识别隐含前提、评价局限性 | 能指出"什么情况下这个结论不成立" |
| 5 | 创造 | 能跨领域迁移、产出原创观点、质疑理论本身 | 能把概念用于完全不同的领域并提出新看法 |

## 按内容类型的目标掌握度

| 内容类型 | 学科示例 | 目标掌握度 | 说明 |
|----------|----------|-----------|------|
| 基础概念 | 经济学基本术语、Python 语法 | 3（应用） | 能解释 + 给例子即可 |
| 理论框架 | 供需模型、IS-LM 模型 | 3（应用） | 能用在具体情境中 |
| 论证分析 | 哲学论证、理论推导 | 4（分析） | 能评价论证质量 |
| 技能实操 | Python 编程、量化策略 | 4（分析） | 能写 + 能解释为什么 |
| 进阶技能 | 策略设计、系统构建 | 5（创造） | 能独立设计和改进 |

## 判定操作指南

**不靠用户自评**，靠输出质量判定。对每个测试回答评估 **4 个维度**：

1. **深度**：回答涉及了几个层面？
   - 仅定义 → 深度 = 浅
   - 定义 + 机制 → 深度 = 中
   - 定义 + 机制 + 应用/反例/局限 → 深度 = 深

2. **结论准确度**：答案的最终结论是否正确？（从"准确度"拆分而来）
   - 无错误 → 准确
   - 有小错但不影响整体 → 基本准确
   - 有方向性误解 → 不准确

3. **推理链完整性**（仅用于机制/传导题）：从前提推出结论的过程中有无跳步或逻辑断裂？
   - 推理链完整，无跳步 → 完整
   - 推理链有微小跳步（漏了 1 个中间环节）→ 有瑕
   - 推理链断裂或因果方向反 → 不完整
   - 非机制题标记为「—」

4. **迁移能力**：能否输出自己的例子？
   - 只复述书上例子 → 无迁移
   - 能给出自己的例子 → 已迁移
   - 能跨领域类比 → 强迁移

**综合判定**：

| 深度 | 结论准确 | 推理链（机制题） | 迁移 | 判定等级 |
|------|----------|-------------------|------|----------|
| 浅 | 准确 | —/完整 | 无 | 2 |
| 中 | 准确 | —/完整 | 无 | 2-3 |
| 中 | 准确 | —/完整 | 已迁移 | 3 |
| 深 | 准确 | 完整 | 无 | 3 |
| 深 | 准确 | —/完整 | 已迁移 | 4 |
| 深 | 准确 | 完整 | 强迁移 | 4-5 |
| 深 | 准确 | 有瑕 | 已迁移 | 3（推理扣分） |
| — | 不准确 | — | — | 1-2（需要补课） |

**边界案例**：
- 用户讲得对但用的是书上的原话 → 最多 2（理解），不能给 3
- 用户举了例子但例子有问题 → 2，需要纠正
- 用户说了很多但不触及核心机制 → 2，需要追问
- **用户答对结论但推理链有瑕** → 至少降 1 级，追加追问，记入 mistakes.md
