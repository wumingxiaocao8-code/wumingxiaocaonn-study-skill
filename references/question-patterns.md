# 苏格拉底式提问模板库

## 提问原则

1. 问题要具体，避免"你怎么看"这种开放式空泛问题
2. 从用户已经接受的前提出发，逐步引出矛盾或深层理解
3. 追问目的是帮助用户把模糊理解变清晰，不是为难用户
4. 如果用户在两轮追问后仍然卡住，切换为直接讲解 + 例子

---

## 按层级的提问模板

### 第 1 层：理解验证

确认用户是否真正理解了核心逻辑，而非只记住术语。

| 模式 | 模板 | 使用场景 |
|------|------|----------|
| 用自己的话说 | "假设你要向一个完全不懂这个的人解释 X，你会怎么说？" | 检验是否真正理解 |
| 举自己的例子 | "给我一个你自己的例子，不要用书上的。" | 检验是否只会复述 |
| 具体化 | "你说的 Y 具体是什么意思？能展开讲讲吗？" | 用户回答太模糊时 |
| 反例检测 | "有没有哪个情况是 X 无法解释的？" | 检验是否理解边界 |

### 第 2 层：深度推理

检验是否理解概念之间的关系和隐含逻辑。

| 模式 | 模板 | 使用场景 |
|------|------|----------|
| 条件变化 | "如果[某个前提条件]变了，你的结论还成立吗？" | 检验理解是否牢固 |
| 观点对立 | "持相反观点的人会怎么反驳你说的？" | 检验能否看到另一面 |
| 关联回溯 | "这和我们之前学过的 X 有什么关系？是补充还是矛盾？" | 检验概念网络 |
| 因果深挖 | "你说 A 导致 B，那中间的具体机制是什么？" | 检验是否理解过程 |
| 极端测试 | "如果把 X 推到极端，会出现什么情况？" | 检验对边界的理解 |

### 第 3 层：批判评价（用于目标掌握度 4-5）

| 模式 | 模板 | 使用场景 |
|------|------|----------|
| 前提审视 | "这个理论建立在什么隐含假设上？如果那个假设不成立呢？" | 分析层级 |
| 局限评价 | "这个解释在什么情况下会失效？为什么？" | 分析层级 |
| 跨域迁移 | "X 的逻辑能不能用到[完全不同的领域]？如果可以，怎么用？" | 创造层级 |
| 理论质疑 | "你觉得 X 理论本身有什么问题？有没有更好的解释？" | 创造层级 |
| 综合建构 | "如果你要改进 X 理论，你会怎么做？" | 创造层级 |

## 按学科的提问模板

### 经济学

- "这个经济模型的简化假设是什么？现实中那些假设成立吗？"
- "如果这个政策实施了，谁会受益、谁会受损？"
- "短期和长期的影响可能有什么不同？"
- "你能不能想一个反例——一个这个理论预测不了的真实经济现象？"

### 哲学

- "这个论证的前提你是否同意？如果不同意前提，结论还站得住吗？"
- "这个立场背后的价值观预设是什么？"
- "如果把同样的推理结构用到另一个问题上，会得出什么结论？那是合理的吗？"
- "这个哲学家的论证和[另一个哲学家]的观点有什么根本分歧？"

### Python / 量化交易

- "这段代码在什么输入下会崩溃？"
- "有没有更简洁的写法？为什么原书选择现在这种写法？"
- "这个策略在什么市场条件下会失效？"
- "如果交易成本增加一倍，这个策略还赚钱吗？"

## 机制追问（过程审计触发后使用）

用于用户答对机制/传导题后，验证推理链完整性。每次仅追加 1 个，不堆叠。

| 场景 | 模板 | 目的 |
|------|------|------|
| 因果链验证 | "你这个结论中间经过哪几步？从 X 到 Y，一步步说。" | 检查是否真理解完整链条 |
| 前提变化 | "如果去掉[某前提]，这个结论还成立吗？为什么？" | 检查是否理解边界条件 |
| 薄弱环节自检 | "你的解释里最薄弱的一环是什么？" | 引导自我审视逻辑缺口 |
| 传导路径验证 | "你说是 A 导致了 B，那 A 是通过什么途径影响 B 的？" | 检查具体机制理解 |
| 反方向验证 | "如果因果方向反过来，有没有可能也说得通？" | 检查是否混淆因果方向 |

## 追问节奏控制

- 第 1 次追问后用户回答得好 → 进入下一层（最多到目标掌握度对应的层级）
- 第 1 次追问后用户卡住 → 给提示，再试一次
- 两次追问后仍然卡住 → 切换为讲解："你卡住的地方是 X，让我换个角度解释……"
- 用户明显疲劳 → 减少追问，记录当前状态，建议休息
