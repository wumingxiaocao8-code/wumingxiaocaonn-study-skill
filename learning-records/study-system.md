# 学习系统规范
最后更新：2026-06-06

本规范用于统一当前项目的学习方式。默认目标不是“读完更多内容”，而是让用户能建立框架、解释机制、迁移到现实问题，并在需要时深入追溯一个话题的历史与当前机制。

## 三种学习模式

| 模式 | 适用场景 | 核心产出 |
|------|----------|----------|
| 读书模式 | 学习一本具体书、章节或段落 | 模块掌握度、错题、复习队列、概念图 |
| 领域研究模式 | 快速进入陌生领域、写报告、搭知识框架 | 领域树、关键问题、横向对比、结果指标 |
| 专题深挖模式 | 用户想详细了解某个话题、机制或历史由来 | 问题树、时间线或机制链、现实问题与争议 |

默认采用“读书 + 领域研究”混合模式：读书时不只跟章节走，还要把章节挂到领域框架、核心问题和输出任务上。

## 读书模式

继续使用 Bloom 掌握学习循环：

```
DIAGNOSE → TEACH → QUESTION → TEST → CORRECT → REVIEW → RECORD
```

新增要求：
- 每个模块必须绑定一个核心问题，不能只写章节标题。
- 每个模块必须有一个输出任务，例如 300 字解释、机制链、对比表、案例判断或概念图。
- 结论正确但推理跳步、机制缺环、边界不清，必须进入 `mistakes.md`，不能因为“答对了”跳过记录。
- 完成模块时记录“解决了什么问题、产出了什么、还有什么边界”。

## 领域研究模式

适用于用户想快速学习陌生领域，或需要给老师、导师、老板交付梳理报告。

启动流程：
1. 先搭领域树：根问题 → 一级模块 → 二级概念 → 关键问题。
2. 建纵向线：从最早、最简单的形态讲到复杂形态，追问每次演变解决了什么旧问题、制造了什么新问题。
3. 建横向线：比较主流流派、方案、国家、技术、案例，说明差异、优缺点和并存原因。
4. 建结果指标线：明确用什么指标判断这个领域，例如 GDP、汇率、财政、就业、贸易、产业结构、资本流动、胜负结果、制度稳定性。
5. 带问题学习：先确定要回答的问题，再读材料、查资料、填框架。
6. 用输出验证：报告、表格、概念图、案例分析或决策备忘录。

粗读材料时只看结构：标题、章节顺序、论证框架、结果指标。细读时再把事实、概念、机制和争议填回框架。

## 专题深挖模式

触发语义包括：“详细讲讲”“深入讲”“从头讲起”“这个机制怎么来的”“历史上怎么形成的”“讲完整一点”。

每次深挖先把大问题拆成 5-8 个小问题，再逐个讲。固定结构：

1. 历史起点：最早从哪里开始，原始形态是什么。
2. 形成缘由：当时要解决什么问题，受哪些条件限制。
3. 演变路径：中间经过哪些阶段，每一阶段为什么变化。
4. 关键转折：哪些事件、人物、制度或技术改变了路径。
5. 当前机制：现在如何运行，核心参与者和约束是什么。
6. 现实问题：现在面临什么矛盾、风险和瓶颈。
7. 争议与开放问题：主流解释、替代解释、证据限制和待验证问题。

讲完后必须输出一张“问题树”“时间线”或“机制链”，并用 1-2 个问题检查用户能否复述核心路径。

## 经济学学习模板

经济学继续以机制链和现实验证为中心。

每个经济学模块至少覆盖：
- 核心概念：这个概念是什么，解决什么问题。
- 机制链：前提 → 中间环节 → 结果。
- 结果指标：GDP、汇率、财政、产业结构、资本流动、就业、贸易、价格、利润、债务等。
- 横向对比：不同国家、政策、学派、阶段或案例的差异。
- 现实验证：用一个现实案例检验理论是否成立。
- 边界条件：什么时候不成立，反对者会怎么说。

测试重点：
- 解释机制。
- 判断边界。
- 横向对比。
- 迁移到现实案例。

## 历史学习模板

历史学习不直接套经济学机制链。零基础历史先建立年代感和叙事骨架，再进入结构分析。

每个历史模块统一拆成六层：

| 层级 | 学什么 | 检查问题 |
|------|--------|----------|
| 时间线 | 关键年代、阶段划分、转折点 | 能不能按顺序讲清楚？ |
| 空间地图 | 地理范围、政权边界、交通路线、资源位置、战争或贸易通道 | 事件发生在什么空间结构里？ |
| 人物与群体 | 皇帝、官僚、军队、士人、商人、农民、外部势力 | 谁推动，谁阻碍，谁承受代价？ |
| 制度与结构 | 土地、税收、军制、官僚体系、财政、宗教、贸易、技术 | 长期约束是什么？ |
| 事件过程 | 起因、导火索、过程、转折、结果 | 不是只背结论，能不能讲过程？ |
| 解释与争议 | 主流解释、替代解释、史料限制、后世影响 | 为什么这样解释，边界在哪里？ |

历史模块的讲解顺序：

```
发生了什么 → 为什么发生 → 怎么发展 → 谁推动/阻碍 → 结果是什么 → 后世影响
```

历史测试题必须覆盖：
- 时间顺序复述。
- 因果链解释。
- 人物选择与制度约束。
- 阶段比较。
- 反事实边界：“如果某个条件改变，结果可能如何不同？”

## 复习与验收

复习不再只按模块机械推进。已完成多本书后，优先按概念簇复习：
- 中心与外围
- 全球化
- 汇率制度
- 美元霸权
- 人民币国际化
- 产业政策与资本

验收标准：
- 能复述一个话题的起点、路径、当前机制和核心问题。
- 能画出一条因果链、机制链或时间线。
- 能指出至少一个争议、边界或替代解释。
- 能把旧书概念迁移到新案例，而不是只背作者结论。
