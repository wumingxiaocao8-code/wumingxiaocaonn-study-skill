## 4. sessions/YYYY-MM-DD.md

单次学习会话记录。位于 `reading/learning-records/<书名>/sessions/YYYY-MM-DD.md`。

```markdown
# 学习会话 2026-05-05
书籍：《货币金融学》
模块：M3 金融体系概览
模式：学习模式
时长：约 35 分钟

## 诊断结果
- 问题 1 "什么是金融中介"：正确（等级 3）
- 问题 2 "直接融资和间接融资的区别"：模糊（等级 2）
- 问题 3 "信息不对称在金融中的作用"：错误（等级 1）
→ 标记重点：直接 vs 间接融资、信息不对称

## 教学内容
1. 金融体系的三大功能：资源配置、风险管理、支付清算
2. 直接融资 vs 间接融资的本质区别：信息处理方式不同
3. 信息不对称的两种形式：逆向选择（事前）、道德风险（事后）

## 测试表现
| 题号 | 题型标记 | 类型 | 结论 | 推理质量 | 扣分点 | 需复测 |
|------|----------|------|------|----------|--------|--------|
| 1 | 定义 | 解释题 | ✅ | — | — | 否 |
| 2 | 机制 | 传导题 | ✅ | 完整 | — | 否 |
| 3 | 机制 | 因果题 | ✅ | 跳步 | 遗漏"信心危机→抛售"环节 | 是 |

## 推理备注
- 题 3 追问："如果不发生信心危机，美元还会贬值吗？" → 用户能正确补全，推理链过关
- 题 1：用户给了自己的例子

## 纠错
- 题 3 跳步：特里芬难题→信心危机→各国减持→美元贬值，缺了"信心危机"环节
- 锚点："信心是货币的命根子"

## 掌握度评定
- 诊断：2 → 教学后测试：3（达到目标，但机制题推理扣分，建议复习时重点复测）

## 本模块覆盖机制
- 特里芬难题 / 美元信心机制 / 浮动汇率传导

## 下次复习：2026-05-08
```

货币 → 利率 → 信贷配给 → 投资决策 → 总需求 → 经济周期

## 9. <书名>聊天记录.md

完整对话流归档。位于 `reading/learning-records/<书名>/<书名>聊天记录.md`。

**由「整理本次学习记录」命令触发**，每次追加一个会话节。不修改已有内容。

**包含内容**：本次会话中用户和 Claude 的所有文本输出（包括教学、提问、回答、推理过程、讨论、表格等）。跳过工具调用、文件读写结果、thinking 等非对话内容。

**格式**：

```markdown
# 会话 2026-05-07

## 教学/讨论内容（按模块或主题分段）

小草：用户发言原文
Claude 的回复原文，保留表格、代码块、列表等格式。

小草：用户的下一句话
Claude 的回复。
```

**规则**：
- 用户发言以 `小草：` 开头（如用户有不同称呼则用对应称呼）
- Claude 回复直接写，不带前缀
- 保持原有 Markdown 格式（表格、代码块、列表、加粗等全部保留）
- 每个会话节以 `# 会话 YYYY-MM-DD` 开头
- 追加到文件末尾，不修改已有内容
- 如果 `reading/learning-records/<书名>/` 下尚不存在此文件，则新建

## 所有档案文件的写入规则

1. 写入前确认 `reading/learning-records/` 目录存在
2. 更新已有文件时必须先 Read 原内容，用 Edit 做精确修改
3. 每次写入后用 Read 验证写入结果
4. 所有日期使用 YYYY-MM-DD 格式
