# 测试场景

以下是 Skill 应能正确处理的测试场景。每个场景包含触发语句、预期步骤序列、预期文件操作。

---

## 场景 1：开始学习这本书

**触发**：
```
用户：开始学习这本书
（当前目录：reading/货币金融学/，内含 货币金融学.md）
```

**Skill 预期行为**：
1. 识别当前目录下存在 md 源文件
2. 提示当前没有学习档案，将初始化
3. 读取 md 文件前 200 行，分析目录结构
4. 输出模块拆分建议（按 module-splitting.md 规则）
5. 等待用户确认拆分方案
6. 用户确认后，创建 `reading/learning-records/货币金融学/` 目录
7. 写入 module-plan.md、book-index.md、current-state.md
8. 更新或创建 global-index.md
9. 进入模块 1 的诊断阶段，出 3-5 道题

**预期文件操作**：
- 创建 `reading/learning-records/货币金融学/module-plan.md`
- 创建 `reading/learning-records/货币金融学/book-index.md`
- 创建 `reading/learning-records/货币金融学/current-state.md`
- 创建或更新 `reading/learning-records/global-index.md`

---

## 场景 2：继续学习

**触发**：
```
（新会话）
用户：继续学习
```

**Skill 预期行为**：
1. 读取 `reading/learning-records/global-index.md` 找进行中的书
2. 读取对应书的 `current-state.md`
3. 读取 `book-index.md`
4. 读取 `module-plan.md`
5. 读取最近一次 session 文件
6. 读取当前模块对应的源文本
7. 输出恢复摘要（格式见下）
8. 检查 review-queue.md 是否有到期项
9. 建议下一步动作，等待用户选择

**恢复摘要格式**：
```
欢迎回来。当前学习状态：

📖 《货币金融学》
   进度：5/18 模块（27.8%）
   当前：M3 金融体系概览（教学 2/3 完成）
   上次：2026-05-05，掌握度 2→3

🔔 到期复习：M2（已到期 1 天）

建议：先复习 M2，再继续 M3。你觉得呢？
```

## 场景 3：快速解释概念

**触发**：
```
用户：快速解释一下什么是"流动性陷阱"，不要走完整教学流程
```

**Skill 预期行为**：
1. 识别"快速解释"触发快速答疑模式
2. 问一个判断理解的问题："关于流动性陷阱，你目前了解到什么程度？"
3. 根据用户回答，用微循环：判断已有理解 → 解释 → 追问 1-2 层
4. 如果用户能清晰复述，结束
5. 不写入 session 文件（除非用户追问中表现出深度兴趣）
6. 不触发完整教学循环

**不应发生**：
- 出模块测试题
- 写入大量记录文件
- 走 7 步完整循环

---

## 场景 4：整理本次学习记录

**触发**：
```
（在学习了约 30 分钟后）
用户：整理本次学习记录
```

**Skill 预期行为**：
1. 回顾本次对话中的学习内容
2. 确认当前模块、测试结果、纠错内容
3. 创建 `sessions/2026-05-05.md`
4. 更新 `module-plan.md`（状态、掌握度、复习日期）
5. 如果有错题 → 更新 `mistakes.md`
6. 如果有新概念 → 更新 `concept-map.md`
7. 更新 `review-queue.md`
8. 更新 `book-index.md`
9. 更新 `current-state.md`
10. 更新 `global-index.md`

**预期文件操作**：
- 写入或更新上述 7 个文件
- 不修改书籍源文件

---

## 场景 5：学习 Python / 量化代码

**触发**：
```
用户：学习 M7（包含 Python 代码的量化策略模块）
```

**Skill 预期行为**：
1. 进入学习模式，走完整循环
2. 诊断阶段如常
3. 教学阶段：对每个代码块执行逐行解释
   - 每行标注：业务逻辑 + Python 原理
   - 变量类型、对象关系、函数调用顺序
4. 测试阶段：
   - 至少 1 道代码题（写变体）
   - 至少 1 道解释题（"为什么第 Y 行这样写"）
5. 纠错：必须给出错因 + 正确代码 + 改进原理
6. 量化策略额外覆盖：市场条件、参数敏感度、交易成本、风险

**不允许**：
- 只给代码不逐行解释
- 说"这里很简单就不展开了"
- 跳过练习环节

---

## 场景 6：复习到期内容

**触发**：
```
用户：复习今天到期内容
```

**Skill 预期行为**：
1. 读取 `review-queue.md`
2. 找出所有到期项
3. 对每项：出 2-3 道重测题
4. 全对 → 掌握度不变/提升，间隔延长一档
5. 有错 → 掌握度下调，纠错，间隔降一档
6. 更新 `review-queue.md` 和 module-plan.md

---

## 场景 7：模式切换

**触发**：
```
（在学习模式中途）
用户：等一下，这里我不太懂，直接给我解释一下
```

**Skill 预期行为**：
1. 立即暂停当前教学流程
2. 切换到快速答疑模式处理用户的问题
3. 处理完毕后："刚才你问的 X 我解释清楚了。要继续模块 3 的教学吗？"
4. 等待用户确认后回到学习模式

---

## 场景 8：跨书关联

**触发**：
```
用户：跨书关联"边际效用"这个概念
```

**Skill 预期行为**：
1. 用 Grep 在所有 `learning-records/*/concept-map.md` 和 `sessions/` 中搜索"边际效用"
2. 找出所有相关记录
3. 输出关联报告：哪些书、哪些模块涉及、掌握度如何、概念之间的异同
4. 询问是否需要写入当前书的 concept-map.md
