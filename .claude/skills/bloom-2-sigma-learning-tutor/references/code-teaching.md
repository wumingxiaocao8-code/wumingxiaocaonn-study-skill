# Python / 量化交易 / 代码教学规则

## 触发条件

当前模块包含代码时（Python、量化策略、算法等），以下规则自动生效。

---

## 逐行解释规则

**每个代码块必须逐行解释**，不允许概括式描述"这段代码做了 X"。

**解释格式**：

```python
def calculate_sharpe(returns, risk_free=0.02):
    excess = returns - risk_free / 252      # ← 这行
    return excess.mean() / excess.std()      # ← 这行
```

每行的解释必须包含：

```
行 x：`<代码>`
  - 业务逻辑：这行在做什么，为什么这一步是必要的
  - Python 原理：变量类型（输入/输出类型是什么）、运算符优先级、对象关系、
    函数调用顺序、是否有副作用
  - 注意点：容易出错的地方、性能考量、Python 特有的行为
```

**不要求每行都写完整三段**——简单赋值的"业务逻辑"可以一句话。但核心逻辑行（运算、函数调用、控制流、数据转换）必须三段齐全。

---

## 代码教学流程

1. **先问**："这段代码你读到什么程度？试试解释一下？"
2. **听用户解释**，标记他说对的和说错的部分
3. **逐行讲解**，重点讲用户没讲清楚的行
4. **出变体题**：改需求、改输入、改边界条件，让用户自己写
5. **验证**：用户贴出代码后，逐行审查——不是只给 pass/fail，而是指出每行对不对、为什么

---

## 练习出题规则

**基础练习**（目标掌握度 3）：
- 改参数："同样的逻辑，如果输入变成 X，你会怎么改？"
- 改场景："把这个函数改成能处理 Y 情况的版本"

**进阶练习**（目标掌握度 4）：
- 找漏洞："这段代码在什么输入下会崩溃？请写出来并修复"
- 优化："有没有更简洁/更快的写法？为什么原书选择现在这种？"

**高级练习**（目标掌握度 5）：
- 独立设计："请设计一个函数/策略实现 X 功能"
- 跨域迁移："把 A 策略的逻辑用到 B 市场上，需要做什么调整？"

---

## 量化交易特殊规则

量化策略教学时必须额外覆盖：

1. **策略逻辑**：这个策略的核心假设是什么？它赚的是什么钱？
2. **市场条件**：什么市场环境下有效？什么环境下会失效？
3. **参数敏感度**：参数变化对策略表现的影响有多大？（过拟合风险）
4. **交易成本**：如果把滑点、手续费、冲击成本加进去，还赚钱吗？
5. **风险检查**：最大回撤预测、尾部风险、杠杆风险

---

## 纠错强化

代码纠错必须给出：

1. **错因**：具体哪一行、哪一个概念理解错了
2. **正确代码**：给出修正后的代码
3. **改进原理**：为什么这样改是对的（不是"这样写更好"，而是"因为 Python 的 X 机制导致原来的写法会 Y"）
4. **对比示例**（如果适用）：展示正确和错误的运行结果差异

---

## 不允许的行为

- 不允许只给代码不逐行解释
- 不允许说"这里很简单就不展开了"
- 不允许只指出错误不给正确代码
- 不允许跳过练习环节（代码学习必须动手写）
