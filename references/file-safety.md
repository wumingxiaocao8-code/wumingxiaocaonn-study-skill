# 文件操作安全规则

## 写入范围限制

### 路径基准

本项目工作区根目录通常就是 `reading`。实际读写路径以项目根为基准，不得在项目根下重复添加 `reading/` 前缀。写入前必须解析绝对路径，并确认目标仍在项目根内。

### 学习记录：仅限 learning-records/

所有学习档案（global-index、book-index、module-plan、sessions、mistakes、review-queue、concept-map、current-state）只能写入 `learning-records/` 及其子目录。

### 书籍源文件：在用户确认后可写入当前书籍目录

书籍源文件的拆分结果只能写入用户确认的项目内书籍目录，例如 `economic/<书名>/chapters/` 或 `history/<书名>/chapters/`。
写入前必须：
1. 向用户说明将要创建的目录和文件
2. 获得用户确认
3. 不得覆盖已有同名文件（除非用户明确同意）

### 绝对禁止

- 写入项目外的任何路径
- 写入 .claude/ 目录（Skill 自身文件除外）
- 修改或删除用户的原始书籍文件（epub/md 源文件）

---

## 写入前规则

### 1. 验证目标路径

每次写入前确认路径在允许范围内（`learning-records/` 或用户确认的项目内书籍目录）。

### 2. 先读后写

修改已有文件前必须先用 Read 工具读取当前内容。不允许盲写。

### 3. 优先用 Edit，避免全量覆盖

修改已有文件使用 Edit 工具做精确替换。如果必须全量覆盖（如新建文件、模板化输出），使用 Write 工具。

### 4. 大改前备份或追加

- 修正学习记录时，优先在 `sessions/` 下新增一条会话记录，不改写历史记录
- 如果必须改写（如修正模块拆分方案），告知用户修改范围并等确认
- 改写前将当前内容复制到同名 `.bak` 文件（仅对关键文件如 module-plan.md、book-index.md）

### 5. 写入后验证

每次写入后用 Read 验证写入结果是否正确。

## 文件命名规范

| 文件类型 | 命名规则 | 示例 |
|----------|----------|------|
| Session 文件 | YYYY-MM-DD.md | 2026-05-05.md |
| 章节文件 | chapter-NN.md 或章节名.md | chapter-03.md |
| 备份文件 | 原文件名.bak.YYYY-MM-DD | module-plan.md.bak.2026-05-05 |

日期统一使用 YYYY-MM-DD 格式。

---

## 错误恢复

如果写入操作失败：
1. 确认目标目录存在（不存在则创建）
2. 检查文件是否被其他进程占用
3. 重试一次
4. 仍失败则告知用户无法写入，在对话中保留内容副本，等待用户手动处理

---

## 禁止操作清单

| 禁止操作 | 说明 |
|----------|------|
| rm / delete | 不得删除用户的任何文件 |
| 覆盖源文件 | 不得覆盖 epub/md 源文件 |
| 项目外写入 | 不得在项目根目录外创建或修改文件 |
| 无提示覆盖 | 不得在用户不知情的情况下覆盖已有内容 |
| 批量操作 | 不得一次性大范围修改多个文件而不逐一说明 |
