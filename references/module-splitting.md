# 模块拆分规则

## 拆分原则

1. 每个模块应是一个可在一小时内完成教学+测试的单元
2. 优先保持主题完整性——不把紧密关联的概念拆到两个模块
3. 模块之间应有清晰的概念边界

---

## 字数阈值标准

以下阈值基于普通阅读速度（300 字/分钟）和教学开销（教学 2-3 倍阅读时间）计算：

| 条件 | 字数 | 页数估算 | 动作 |
|------|------|----------|------|
| 短章 | < 3000 字 | < 5 页 | 与相邻章节合并 |
| 标准 | 3000-8000 字 | 5-13 页 | 保持为一个模块 |
| 长章 | 8000-12000 字 | 13-20 页 | 考虑拆分 |
| 超长 | > 12000 字 | > 20 页 | 必须拆分 |

---

## 拆分方法论

### 1. 按章拆分（默认）

书籍有清晰章节结构时，一章 = 一个模块。

### 2. 按节拆分（长章节 > 12000 字）

章节内部有 §1、§2 等结构时，按节边界拆分。
拆分后每个子模块控制在 4000-8000 字。

### 3. 合并短章（< 3000 字）

相邻短章如果主题连续（属于同一主题域），合并为 1 个模块。
合并后总字数控制在 5000-8000 字。

### 4. 按概念群拆分（理论密集章）

适用条件：章节包含 ≥ 4 个独立核心概念，每个概念都有足够的复杂度。

拆分单位不是"字数均分"而是"概念自然聚类"：
- 找出章内所有的核心概念
- 把相关概念聚类（如"流动性偏好 + 利率期限结构 + 收益率曲线"）
- 每个聚类 = 1 个模块

### 5. 按技能点拆分（Python / 量化内容）

每个可独立练习和验证的技能 = 1 个模块。

例如 Python 书：
- "装饰器" = 1 个模块
- "上下文管理器" = 1 个模块
- "元类"可能拆 2 个模块（基础 + 进阶）

## 拆分输出格式

拆分完成后输出以下格式，供用户确认：

```markdown
## 模块划分建议 — 《货币金融学》
全书 24 章 → 建议拆为 18 个模块

| 模块 | 内容 | 源章节 | 字数估算 | 拆分原因 |
|------|------|--------|----------|----------|
| M1 | 货币的本质与职能 | 第1章 | ~4000 | 标准 |
| M2 | 利率决定理论 | 第2-3章 | ~5500 | 短章合并 |
| M3 | 金融体系概览 | 第4章§1-2 | ~6000 | 长章拆分 |
| M4 | 商业银行与货币创造 | 第4章§3-5 | ~8000 | 长章拆分 |
| M5 | 价格弹性 | 第5章§1-2 | ~4500 | 概念群拆分 |
| M6 | 收入弹性与交叉弹性 | 第5章§3-4 | ~5000 | 概念群拆分 |
| ... | ... | ... | ... | ... |

请确认或提出调整。输入"确认"后开始学习模块 1。
```

## 拆分工具说明

### 脚本方式（优先尝试）

使用 `scripts/inspect_book_structure.py`：
- 输入：md 文件路径
- 输出：章节结构 JSON（每章字数、标题层级）
- 脚本失败不影响教学流程

使用 `scripts/split_markdown_book.py`：
- 输入：全书 md 文件路径 + 输出目录
- 输出：按 `##` 标题拆分为独立文件
- 脚本失败不影响教学流程

### 手动方式（脚本不可用时的回退）

脚本运行失败时，Skill 必须回退到手动方式：
1. 用 Read 工具读取目录页或前 200 行，分析 `#` `##` 标题层级
2. 快速扫描各章开头，估算字数
3. 按上述阈值标准给出拆分建议
4. 每次学习时只读取当前章节/模块的文本片段
5. 不得中断教学流程
